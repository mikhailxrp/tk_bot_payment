# Global Definition of Done

Эти критерии применяются к каждому таску проекта (бот + панель).
Разделы, не касающиеся таска, пропускаются целиком.

---

## TypeScript

- [ ] `npm run type-check` проходит без ошибок во всех workspace
- [ ] Нет `any` (только `unknown` при действительно динамическом типе)
- [ ] Zod-схемы для env, webhook-параметров, форм и server actions; типы через `z.infer<>`
- [ ] Возвращаемые типы экспортируемых функций указаны явно
- [ ] `telegram_id` везде BigInt; в местах JSON-сериализации BigInt обработан явно

## Платежи / Робокасса (если таск касается оплаты)

- [ ] Подпись ссылки: `md5(Login:OutSum:InvId:Receipt:Pass1)`; Receipt URL-encoded один раз в подписи, дважды в URL
- [ ] Подпись webhook: `md5(OutSum:InvId:Pass2)`, сравнение case-insensitive
- [ ] `OutSum` — строка с двумя знаками после точки, формат одинаков в ссылке и в проверке
- [ ] Начисление ТОЛЬКО по Result URL; Success URL ничего не начисляет
- [ ] Ответ на валидный webhook — строго `OK{InvId}` plain text
- [ ] Идемпотентность: повторный webhook по `PAID`-платежу → `OK` без изменений
- [ ] Продление + Payment.PAID + unmute + сброс флагов напоминаний — одна `prisma.$transaction`
- [ ] `sno`/`tax` в Receipt берутся из настроек/env, не захардкожены
- [ ] `IsTest` управляется env, в проде выключен

## Telegram API (если таск касается группы/рассылок)

- [ ] Блокировка = `restrictChatMember(can_send_messages: false)`; `banChatMember` не используется нигде
- [ ] Unmute возвращает права по умолчанию супергруппы
- [ ] Invite-ссылки: `createChatInviteLink({ member_limit: 1 })`, одноразовые, создаются в момент оплаты, не переиспользуются
- [ ] Рассылки только через `services/notify.ts`: ≤ 25 msg/sec
- [ ] Ошибка 403 (юзер заблокировал бота) логируется и НЕ роняет джобу/цикл

## Cron / конкурентность (если таск касается dailyCheck или фоновых задач)

- [ ] Таймзона задана явно (`Europe/Moscow`)
- [ ] Ручной запуск из /admin и cron не могут работать одновременно: MySQL `GET_LOCK()/RELEASE_LOCK()` (НЕ pg_advisory — у нас MySQL)
- [ ] Смена статусов — атомарным guard: `updateMany({ where: { status: 'ACTIVE', expiresAt: { lt: now } } })`, а не «прочитать → проверить в коде → записать»
- [ ] Флаги напоминаний (`reminderSentAt`, `lastMutedRemindAt`) исключают повторную отправку и сбрасываются при оплате
- [ ] Граничные даты выборок покрыты тестом (ровно 3 дня; 9/10/20 дней mute)

## Панель: UI / UX

- [ ] Страница открывается без ошибок консоли
- [ ] Loading states (Suspense или `loading.tsx`), error states (`error.tsx`)
- [ ] Пустые состояния (нет пользователей, нет платежей) обработаны явно
- [ ] Вёрстка не ломается на 1024×700

## Панель: формы и валидация

- [ ] Валидация Zod на клиенте и повторно в server action
- [ ] Ошибки полей подсвечиваются; после успеха — редирект или toast

## Панель: Server Actions и безопасность

- [ ] КАЖДЫЙ server action: проверка сессии через `auth()` + Zod-валидация входа — первым делом
- [ ] Доступ только для записей `Admin` (проверка не на клиенте, а в action/route)
- [ ] Незалогиненный не открывает защищённые страницы (guard в layout/middleware)
- [ ] `INTERNAL_API_TOKEN`, `BOT_TOKEN`, пароли Робокассы никогда не попадают на клиент (только `NEXT_PUBLIC_*` допустимы в клиентском коде)
- [ ] Клиенту не возвращаются `passwordHash` и полные токены
- [ ] mute/unmute из панели — только через internal API бота с Bearer-токеном, панель не вызывает Telegram API напрямую

## Данные

- [ ] Изменение схемы = миграция Prisma (`migrate dev`), схема только в `packages/db`
- [ ] Настройки (`Setting`) читаются из БД при каждом использовании, не кэшируются в константах
- [ ] После операции запись реально появляется/меняется в БД (проверка через Prisma Studio)

## Код

- [ ] `npm run build` успешен во всех workspace
- [ ] `npm run lint` без ошибок; нет неиспользуемых импортов
- [ ] Нет warnings в терминале dev-сервера
- [ ] Секреты только через `config.ts` (zod); невалидный env роняет старт
