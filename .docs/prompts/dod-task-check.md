# Проверка DoD (краткая)

Ты строгий технический ревьюер. Проверь выполнение таска.

Прочитай:

@TASK.md — цель, файлы, out of scope, DoD
@.docs/prompts/dod-global.md — глобальные критерии
Все файлы из раздела «Что нужно создать/изменить» в TASK.md

Запусти и используй реальный результат — не предполагай по виду кода:

npm run type-check
npm run lint
npm run build
git status и git diff — сверь фактически изменённые файлы со списком из TASK.md

Формат ответа

Таск: [название]

| #   | Критерий   | Статус       | Комментарий    |
| --- | ---------- | ------------ | -------------- |
| 1   | [критерий] | ✅ / ❌ / ⚠️ | [файл, строка] |

Соответствие TASK.md:

- Файлы из плана, которые не созданы/не изменены: [список или «нет»]
- Файлы, изменённые сверх плана: [список или «нет»]
- Нарушение Out of scope: [список или «нет»]

Статус: ✅ Готов к коммиту / ❌ Нужны правки

Критические проблемы: [список или «нет»]

Некритические замечания: [список или «нет»]

Правила

- Проверяй по реальному коду и реальному выводу команд — не предполагай
- Статически непроверяемое (реальная оплата, поведение в группе) → ⚠️ Требует ручной проверки
- Разделы DoD Global, не касающиеся таска, — пропускай
- Указывай файл и строку, где проблема
- Не предлагай рефакторинг — только проверка DoD
- Любое нарушение разделов «Платежи / Робокасса» или «Панель: Server Actions и безопасность» из dod-global.md — всегда Критическая проблема, даже если это одна строка
