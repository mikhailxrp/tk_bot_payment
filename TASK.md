## ⚠️ Точки риска при реализации

- **Ручную проверку я не могу выполнить сам** — нужен реальный `BOT_TOKEN`, тестовая
  супергруппа, запущенный процесс бота и реальная MySQL. Риск — засчитать таск закрытым по
  одному только зелёному `npm test`, не выполнив фактическую ручную проверку из DoD.
- **Day-guard планировщика хранится в памяти процесса.** Проверка «смена `cron_time`
  подхватывается без рестарта» должна выполняться на уже запущенном процессе бота, не
  перезапуская его между правкой `Setting.cron_time` и следующим тиком — иначе тест ничего не
  доказывает.
- **`test-webhook.ts` бьёт по локально поднятому Fastify**, а не по реальному Result URL
  Робокассы — годится для проверки цепочки «webhook → unmute», но не заменяет более раннюю
  ручную проверку подписи с `IsTest=1` (уже сделана в Фазе 3); не путать эти две проверки.
- **Ложное чувство полноты от 94 проходящих тестов.** Часть DoD фазы 6 (реальный Telegram,
  реальное течение календарных суток МСК) принципиально не автоматизируется мокапами — зелёный
  `npm test` не заменяет пункты ручной проверки в DoD.
