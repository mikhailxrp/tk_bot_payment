# TASK: services/notify.ts — рассылка админам с троттлингом

## Фаза

Phase 5 — Выдача доступа в группы

## Статус

🔄 В работе

## Цель

Есть переиспользуемый механизм рассылки сообщений админам из `apps/bot/src/services/notify.ts`:
троттлинг ≤25 msg/sec, ошибки отправки одному получателю (403 и другие) логируются и не
прерывают рассылку остальным. Функция готова к использованию в таске «уведомление об оплате»
(Task 5.2) и переиспользуема в Фазе 7 (напоминания пользователям) без изменений сигнатуры.

## Что нужно создать/изменить

- `apps/bot/src/services/notify.ts` (bot, новый) — экспортирует:
  - генеричную throttled-примитив-функцию отправки текстового сообщения по списку `chatId: bigint[]`
    с троттлингом ≤25 msg/sec (переиспользуема в Фазе 7 без изменений);
  - `notifyAdmins(bot, text)` — читает все `Admin.telegramId` из Prisma, шлёт через эту

- Retry для неотправленных сообщений

## ⚠️ Точки риска при реализации

- **BigInt → chat_id.** `Admin.telegramId` — BigInt; grammY `sendMessage` принимает
  `number | string`, но не `bigint`. Явный `.toString()` перед вызовом API и перед логированием
  (сырой BigInt в `logger.info({...})` уронит сериализацию).
- **Троттлинг ≤25/sec.** Наивный `Promise.all` по всем админам выглядит рабочим на малом числе
  админов, но не троттлит вообще. Нужен реальный последовательный делей (батчи по 25 + пауза,
  либо интервал ~40ms между отправками).
- **Тестируемость троттлинга.** Обычный `await sleep(ms)` делает тест либо реально медленным,
  либо непроверяемым детерминированно. Нужны fake timers (`vi.useFakeTimers`) или инъекция
  функции задержки.
- **Обработка ошибок — по каждому получателю, а не по всей рассылке.** `try/catch` вокруг всего
  цикла остановит рассылку на первом же 403. Нужен `try/catch` внутри цикла на каждое сообщение;
  та же catch-and-continue логика должна работать и для прочих ошибок (`HttpError`, другие
  коды `GrammyError`), иначе сеть на одном сообщении уронит всю рассылку.
- **Пустой список админов** (ещё не засижен ни один `Admin`, кроме дефолтного) — функция не
  должна падать/делить на ноль на пустом массиве.
