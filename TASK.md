# TASK: Middleware isAdmin + skeleton /admin

## Фаза

Phase 2 — Базовый бот: /start и меню подписки

## Статус

✅ Готово

## Цель

Появляется middleware `isAdmin`, проверяющий право доступа к админ-функциям бота через таблицу
`Admin` в БД (BigInt-сравнение `telegramId`, запрос при каждом вызове — без сравнения с
`config.ADMIN_ID`). Команда `/admin` доступна только администраторам и отвечает заглушкой;
не-админ получает явный отказ, бот не падает. Ручная проверка подписок, сводка и ссылка на
панель — в следующих фазах.

## Что нужно создать/изменить

- `apps/bot/src/bot/middleware/isAdmin.ts` (bot) — middleware: приводит `ctx.from.id` к `BigInt`,
  ищет запись в `Admin.telegramId` через Prisma; если найдена — `next()`, иначе отвечает
  сообщением-отказом и не пропускает дальше. Обрабатывает случай `ctx.from === undefined`.
- `apps/bot/src/bot/handlers/admin.ts` (bot) — `handleAdmin`: заглушка-ответ администратору
  (текст вида «Админ-панель бота: функциональность появится в следующих фазах»).
- `apps/bot/src/bot/bot.ts` (bot) — регистрация `bot.command('admin', isAdmin, handleAdmin)`
  (middleware подключается точечно к `/admin`, не глобально через `bot.use()`).

Изменений в `packages/db` не требуется — модель `Admin` уже есть (Фаза 1).

## Out of scope

- Ручная проверка подписок (Фаза 5)
- Сводка платежей/статистики (Фаза 6)
- Ссылка на веб-панель (осмысленна только после Фазы 7)

## ⚠️ Точки риска при реализации

- **Middleware не должен быть глобальным.** Подключение через `bot.use()` заставит каждый
  апдейт (включая `/start` не-админов) идти через проверку и рискует случайно заблокировать
  остальные хендлеры, если `next()` не вызван корректно. Подключать только к `/admin`.
- **`ctx.from` может быть `undefined`** — обращение к `ctx.from.id` без проверки уронит хендлер.
  Нужен ранний `return`/отказ, как уже сделано в `handleStart`.
- **BigInt-сравнение.** `ctx.from.id` — `number`, `Admin.telegramId` — `BigInt`; нужно явно
  `BigInt(ctx.from.id)` перед запросом, иначе рантайм-ошибка или несовпадение типов.
- **Отказ должен быть явным ответом**, а не молчаливым игнором — иначе критерий «не-админ
  получает отказ» не проверяем; при этом ошибка Prisma/Telegram не должна пробрасываться как
  необработанное исключение (есть страховка `bot.catch` в `index.ts`, но сам хендлер должен
  вести себя предсказуемо).
